# Knowledge Learning Loop

围绕一个主题，完成从研究探索、交互学习到知识卡片沉淀的完整闭环。

## 核心原则

- **不以一次性回答为目标**，而是将学习过程沉淀为可复用的知识卡片
- **先定位再探索**，避免无边界扩展
- **先提纲再学习**，与用户确认目标后再推进
- **先草稿再写入**，确认后再落卡

## 主流程

### 1. 语义理解与知识树定位

从用户问题中识别本次学习主题，并优先匹配到现有知识树中的位置。

**执行步骤：**
1. 使用 `memory_search` 搜索已有知识库
2. 使用 `read` 读取知识地图（`知识地图.md`）
3. 判断：补充已有节点 / 新建卡片 / 扩展新分支

### 2. 探索阶段

围绕已定位的知识树位置开展必要探索。

**信息源优先级：**
1. 已有知识库（`memory_search` + `read`）
2. 本地源码仓库（`read` + `exec`）
3. 互联网搜索（`web_search` + `web_fetch`）

**执行规则：**
- 优先检查现有知识树中的相关节点，避免重复研究
- 当已有知识库不足时，再扩展到本地源码
- 只有当本地仍不足时，才扩展到互联网搜索
- 产出是面向后续学习的研究摘要，不是直接生成卡片

### 3. 学习提纲与交互式学习

基于研究摘要生成学习提纲，先与用户达成共识。

**执行步骤：**
1. 生成学习提纲
2. **与用户确认**：学习目标、范围、提纲安排
3. 按提纲逐项推进学习
4. 默认采用苏格拉底式引导，通过提问帮助用户澄清理解
5. 提纲最后一项为**快速复习回顾**
6. 全部完成后，进入知识卡片沉淀阶段

### 4. 卡片更新与新建决策

在写入前判断当前内容应如何处理：

| 情况 | 处理方式 |
|------|----------|
| 与已有卡片高度相关 | 补充或更新已有卡片 |
| 属于现有分支下的新知识点 | 在该分支下新建卡片 |
| 无法归入现有分支，具备独立价值 | 扩展为新分支 |

### 5. 知识卡片沉淀

按固定结构生成卡片草稿，经用户确认后写入。

**执行步骤：**
1. 按模板生成卡片草稿（见 [references/card-template.md](references/card-template.md)）
2. **与用户确认**
3. 使用 `write` 或 `edit` 写入知识树
4. 更新知识地图

## 异常处理

| 情况 | 处理 |
|------|------|
| 无法定位到知识树 | 先做最小澄清，再决定方向 |
| 探索结果不足以支撑学习 | 继续补充探索，不进入卡片沉淀 |
| 学习目标未达成 | 不提前生成卡片 |
| 与已有卡片关系不清 | 先确认是补充还是新建 |
